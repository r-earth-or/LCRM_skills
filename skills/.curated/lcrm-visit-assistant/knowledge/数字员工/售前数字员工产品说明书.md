# 售前数字员工 — 产品说明书

> 面向销售、合作伙伴、潜在客户的产品介绍文档
> 最后更新：2026-02-24

---

## 一句话定位

**让普通销售拥有资深售前的能力，让售前团队的产能翻倍。**

---

## 产品是什么

售前数字员工是一个能独立完成售前工作的 AI 员工。

它不是问答机器人，不是搜索工具，而是能像资深售前工程师一样：读懂客户需求、匹配产品配置、生成方案报价、处理招投标文件。

**核心赛道**：模块化配置与选型（Complex Configuration & Selection）

适合的场景是：产品本身是标准组件，但客户需求非标，需要根据场景做复杂配置组合。这类工作规则明确但繁琐，人工容易出错，AI 恰好擅长。

---

## 解决什么问题

### 企业的真实痛点

| 痛点 | 具体表现 |
|------|---------|
| **售前产能瓶颈** | 销售增长，但售前人员不够，方案响应慢，客户等不起 |
| **配置出错成本高** | 漏配模块、选错型号、算错数量，导致返工、投诉、亏损 |
| **知识依赖老法师** | 产品知识集中在少数资深员工，新人上手慢，老人离职即断层 |
| **标书耗时耗力** | 招投标频繁，每份标书需要售前花几天时间，重复劳动多 |
| **方案质量参差** | 不同销售出的方案质量差异大，客户体验不一致 |

### 数字员工的解法

不是替代售前团队，而是**让每个销售都能调用售前级别的能力**，让资深售前从重复劳动中解放出来做高价值工作。

---

## 5大核心能力

| 能力 | 解决什么 | 典型场景 |
|------|---------|---------|
| **① 深度知识摄入与精准问答** | 销售记不住产品参数，客户问什么都要查 | 万级SKU的产品，销售秒回技术问题 |
| **② 咨询式需求引导** | 客户需求模糊，需要翻译成技术指标 | 从客户描述的使用场景，锁定唯一匹配型号 |
| **③ 确定性逻辑推理与计算** | 配置规则复杂，人工容易漏项出错 | 兼容性校验、参数匹配、BOM自动生成 |
| **④ 强合规与标书生成** | 招投标耗时，偏离表、技术响应反复改 | 解析招标文件，自动生成技术响应和偏离表 |
| **⑤ 方案与报价闭环** | 配置清单冗长，跨部门核对耗时 | 完整方案包输出，含配置清单、报价、说明 |

## 目标客户画像

### 高匹配客户（强烈推荐）

同时满足以下特征：

| 维度 | 标准 |
|------|------|
| **产品特征** | SKU数量级千级以上，或产品型号数百但配置组合极多 |
| **销售模式** | B2B直销为主，客单价10万+，销售周期长 |
| **售前特征** | 有专职售前团队，售前工作占销售周期30%以上 |
| **招投标** | 年招投标频繁，标书制作是常规工作 |
| **行业** | 工控/安防/ICT/光通信/机械设备/电力设备/医疗器械 |

### 典型行业举例

| 行业 | 为什么匹配 | 典型客户类型 |
|------|-----------|------------|
| **光通信/网络设备** | 标准光模块SKU多，不同网络架构需要不同型号组合，配错即返工 | 光模块厂商、交换机厂商 |
| **工控与驱动** | 变频器/PLC/伺服参数复杂，兼容性校验是核心痛点 | 工控设备厂商 |
| **安防/商显** | 产品线宽，招投标频繁，标书量大 | 安防设备厂商 |
| **工业软件** | 标准模块（MES/ERP/PLM）按工厂需求组合，方案设计复杂 | 工业软件厂商 |
| **医疗器械** | 合规要求高，技术文档复杂，招投标是主要销售方式 | 医疗设备厂商 |
| **服务机器人** | 标准机型+配件，不同场所需要不同配置，渠道商需要快速报价 | 机器人厂商 |

### 低匹配客户（不适合）

出现以下任一特征，售前数字员工价值有限：

- 产品单一标准化（SKU < 100，无配置组合）
- 渠道销售为主，无需复杂售前支持
- 客单价低（< 1万），走量模式
- 无直销团队，依赖经销商

> 低匹配客户可评估 Langtum 平台，按具体业务场景定制 AI 应用。

## 常见异议应对

**"我们自己有售前团队，不需要AI"**
→ 不是替代，是放大。让资深售前做高价值判断，数字员工做重复性配置和文档工作。产能翻倍，不裁人。

**"AI不靠谱，会出错"**
→ 配置选型是确定性逻辑，不是创意工作。规则明确的场景准确率可达95%+，比人工出错率低得多。

**"数据安全怎么保证"**
→ 支持私有化部署，数据不出客户服务器。产品知识库由客户自己维护和控制。

**"我们已经有配置器了"**
→ 配置器解决"按规则选型"，数字员工解决"理解需求→推荐配置→生成方案"的完整链路。两者不冲突，数字员工是配置器的上游。

## 商业模式

| 模式 | 说明 |
|------|------|
| **岗位订阅** | 按数字员工岗位数订阅，约为同等人力成本的 40-50% |
| **效果分润** | 按处理方案数量或节省工时计费（探索中） |
| **试点优先** | 建议从一个区域/一条产品线/一类标书开始试点，验证效果后扩展 |

## 适合的切入场景（销售参考）

优先找到以下任一场景，作为首个突破口：

1. **标书量大**：年招投标50次以上，每次标书制作耗时3天+
2. **新人上手慢**：销售入职后3个月内无法独立报价，依赖老法师
3. **配置出错频繁**：每月有返工或客诉，原因是配错型号或漏配模块
4. **售前产能瓶颈**：销售增长但售前人手不够，方案响应慢影响成单
