# 售前数字员工（PSA）项目

> 语核核心产品线 | 当前阶段：PMF探索期
> 最后更新：2026-02-14

---

## 项目目标

为企业交付能独立完成售前工作的 AI 数字员工，直接替代或辅助中级/高级售前工程师/技术支持岗位。

**核心定位**：模块化配置与选型（Complex Configuration & Selection）—— 在标品和纯定制之间的复杂配置场景。

### 赛道理解（关键认知）

**什么是"模块化配置与选型"？**
- ✅ 产品本身是**标准组件/模块**（不是完全定制）
- ✅ 客户需求**非标**（不同场景需要不同配置组合）
- ✅ 存在**明确但复杂的配置规则**（参数匹配、兼容性校验、业务约束）

**典型场景举例**：
| 行业 | 产品特征 | 配置复杂度 |
|------|---------|-----------|
| 光通信 | 标准光模块SKU（100G/400G/800G），不同网络架构需要不同型号组合 | ⭐⭐⭐⭐⭐ |
| 服务机器人 | 标准机型+配件（充电桩/梯控），不同场所需要不同数量和配置 | ⭐⭐⭐⭐ |
| 工业软件 | 标准模块（MES/ERP/PLM），不同工厂需要不同模块组合 | ⭐⭐⭐⭐ |
| 网络设备 | 标准交换机/路由器，不同网络拓扑需要不同型号和配置 | ⭐⭐⭐⭐⭐ |

**判断客户匹配度的三个标准**：
1. **配置规则明确且复杂**：有大量if-then规则，人工容易遗漏
2. **销售配单易出错**：经常漏配模块、选错型号、算错数量
3. **大量重复性方案制作**：每月几十/上百份方案，耗时且低效

**常见误区**：
- ❌ "产品太标准化，不需要AI" —— 错！标准组件的复杂组合恰恰是AI的优势
- ❌ "配置逻辑简单，Excel就能解决" —— 要验证：是否有隐性规则？是否经常出错？
- ❌ "已有配置器，不需要AI" —— 要对比：配置器是否懂业务规则？是否能解释推荐理由？

---

### 场景分层：核心价值 vs 有价值

**核心价值场景（主赛道）——必须同时满足**：
| 特征 | 说明 | 示例 |
|------|------|------|
| **配置复杂度** | 多组件组合、参数匹配、兼容性校验 | 光模块选型（速率/距离/封装匹配）、工控机配置 |
| **出错成本** | 配错会导致无法交付或客户投诉 | 漏配电源模块、选错接口类型 |
| **高频重复** | 每月几十上百次，人工耗时 | 每单需30分钟-2小时做方案 |

**有价值场景（扩展应用）——可做单点突破**：
| 场景 | 价值 | 与核心赛道的区别 |
|------|------|-----------------|
| **渠道赋能** | 让渠道商快速获得产品知识，减少对总部支持依赖 | 是"知识问答"，不是"配置选型" |
| **招投标辅助** | 自动解析招标文件，生成技术响应 | 是"文档生成"，不是"配置计算" |
| **FAQ应答** | 回答客户技术问题，替代初级售前 | 是"信息检索"，不是"方案设计" |

**关键区分原则**：
- 核心价值场景 = **替代方案设计环节**（最难、价值最高）
- 有价值场景 = **替代信息传递环节**（可行、但价值较低）

**客户评估时**：先判断是否满足核心价值场景的三个条件，不满足则降级为"有价值但非核心"，优先级降低。

## 产品体系

### 主打产品：售前数字员工（PSA）

见上文全部内容。核心定位：模块化配置与选型赛道，目标客户是SKU复杂、招投标频繁的B2B企业。

### 通用平台：Langtum 低代码 Agent 构建平台

- **定位**：企业级 Agentic 数字员工构建平台，无需编程
- **与售前数字员工关系**：Langtum 是底座，售前数字员工是其上的典型应用
- **适用场景**：任何有复杂业务流程、需要降本增效的企业
- **白皮书**：`通用AI Agent开发平台（langtum低代码Agent构建平台产品）/Langtum v1.12.1 产品白皮书.pdf`

#### 5大核心能力

| 能力 | 说明 |
|------|------|
| **强大文档解析** | 支持PDF/Word/Excel/图片等多格式文档自动解析，结构化提取关键信息 |
| **灵活要素提取** | 自定义提取规则，从非结构化文本中精准抽取业务字段 |
| **持久化记忆** | 跨会话保留上下文，支持长期业务知识积累与调用 |
| **灵活工作流** | 可视化编排多步骤业务流程，支持条件分支、循环、并行处理 |
| **企业级权限** | 多租户隔离、角色权限管理、数据安全合规 |

#### 典型应用场景

| 场景 | 业务价值 |
|------|---------|
| **售前辅助** | 自动解析招标文件、生成技术响应、产品知识问答 |
| **供应链管理** | 采购单解析、供应商资质审查、物料匹配 |
| **品牌审查** | 自动检测营销物料合规性，标记违规内容 |
| **产线质检** | 质检报告解析、异常数据提取、质量趋势分析 |

#### 平台架构（5大模块）

| 模块 | 功能 |
|------|------|
| **应用** | 创建和管理 AI 应用，配置对话界面与业务逻辑 |
| **对话** | 多轮对话管理，支持 Chatbot / Multi-Agent 模式 |
| **Agent资源库** | 预置行业 Agent 模板，可直接复用或二次定制 |
| **模型** | 接入主流大模型（GPT/Claude/文心等），统一管理 |
| **设置** | 权限管理、数据安全、系统集成配置 |

### 产品选择逻辑
